What is Git?

Git is a version control system — a tool that tracks every change made to your files over time. Think of it like a save-game system for your code: at any point you can save a "checkpoint" (called a commit), and go back to it later if something breaks.

Git runs entirely on your own computer (it's "local"), so you don't need the internet to use it. It was created in 2005 by Linus Torvalds, the creator of Linux.

What is GitHub?

GitHub is a website that stores your Git projects online ("in the cloud") so you can access them from anywhere and share them with others. If Git is the save-game system, GitHub is like Google Drive for that save data — except built specifically for code.

GitHub adds features on top of Git: pull requests, issue tracking, project boards, and a place for teams (or the whole world) to collaborate on the same codebase.

Uses of Git & GitHub

🗂️

Version Tracking

Keep a complete history of every change to your project, with the ability to undo mistakes.

👥

Team Collaboration

Multiple people can work on the same project at once without overwriting each other's work.

🌐

Open Source

Share your code publicly so anyone in the world can use it, learn from it, or contribute.

📁

Portfolio Building

Showcase your projects to potential employers through a public GitHub profile.

🔄

Backup & Sync

Your code is safely stored online and synced across every device you work from.

🚀

CI/CD & Automation

Automatically test and deploy code whenever changes are pushed to GitHub.

Why Use Git & GitHub?

⏪

Never Lose Work

Every commit is a safe checkpoint — undo mistakes instantly instead of starting over.

🌿

Experiment Safely

Branches let you try new ideas in isolation without breaking the working project.

🤝

Work Together

Teams can work on different features simultaneously, then merge their work together.

🏢

Industry Standard

Almost every tech company uses Git & GitHub — it's an essential skill for any developer.

Problems Git & GitHub Solve

❌ "final_v2_FINAL_reallyfinal.zip" → ✅ Clean commit history with clear messages
❌ Overwriting a teammate's changes by accident → ✅ Merging changes safely, conflict by conflict
❌ Losing all your code when a laptop dies → ✅ Code safely backed up on GitHub's servers
❌ No record of who changed what, or why → ✅ Full history: who, what, when, and why

Basic Git Commands & Workflows

Here are the essential commands every student needs to know, from setting up a project to syncing changes with GitHub.

git init

Creates a brand-new, empty Git repository in your current folder.

git init

git clone

Downloads (copies) an existing repository from GitHub onto your computer.

git clone https://github.com/user/repo.git

git add

Stages changed files, marking them ready to be included in the next commit.

git add filename.txt
git add .   # stage everything

git commit

Saves a checkpoint (snapshot) of your staged changes with a description message.

git commit -m "Add login page"

git push

Uploads your local commits to GitHub, so others (and the cloud) have your latest work.

git push origin main

git pull

Downloads the latest changes from GitHub and merges them into your local copy.

git pull origin main

git branch

Creates (or lists) branches — separate lines of work that don't affect the main project.

git branch new-feature

git checkout / switch

Switches you to a different branch, so you can work on it.

git checkout new-feature
git switch new-feature   # newer syntax

git merge

Combines changes from one branch into another (e.g. feature into main).

git checkout main
git merge new-feature

git status

Shows what's changed, what's staged, and what branch you're currently on.

git status

git log

Displays the history of commits — who changed what, and when.

git log --oneline

git remote

Manages connections between your local repo and remote repos like GitHub.

git remote add origin https://github.com/user/repo.git
